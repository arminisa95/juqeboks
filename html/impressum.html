üîÑ SPA System Architecture

JUKE uses a Single Page Application (SPA) architecture for a seamless, app-like experience. Instead of loading entire pages, the SPA dynamically updates content without full page refreshes.

üè†

Single Entry Point

The entire application runs from index.html. All "pages" are actually templates that get loaded dynamically into the main content area.

üß≠

Hash-Based Routing

Navigation uses URL hashes (#/feed, #/upload, etc.) to manage routes without server requests. The router detects hash changes and loads appropriate content.

‚ö°

Dynamic Content Loading

When you navigate, the SPA fetches HTML templates from the html/ folder and injects them into the main app container - no page reload needed!

üéØ

State Management

User authentication, music player state, and app data persist across navigation. The SPA maintains context while switching between sections.
